## Project Description
本專案旨在建立一套 **商圈輿情分析平台**。

系統將透過 Apify API(Google Maps Extractor) 搜尋特定縣市店家，蒐集店家基本資訊，再利用 Apify API(Google Maps Extractor) 抓取公開評論，經過資料清洗與 AI 分析後，將分析結果儲存至 PostgreSQL，最後提供 Dashboard 視覺化分析與 Threads 自動發文功能。

目標並非單純爬取評論，而是建立一套完整且可持續運行的資料分析 Pipeline，提供使用者快速掌握輿情、潛在問題店家，滿足大眾吃瓜心態。

# 2. Background
目前 Google Maps 雖提供店家資訊與評論，但缺乏：

* 大量資料分析能力
* 具爭議性的評論整理
* AI 摘要分析
* 自動化每日追蹤

使用者若想分析一整個商圈，仍須逐間店家查詢，效率極低。
因此建立一套自動化平台，降低人工搜尋成本，提升資料分析效率。

# 5. Design Principles (開發原則)
1. Contract First Development：所有資料格式須先完成定義，再開始程式開發。
2. Single Source of Truth：所有正式資料皆以 PostgreSQL 為唯一資料來源。
3. Domain-Oriented Design：每個 Domain 僅負責單一職責。
4. MVP First：重要的先做，再逐步擴充功能。
5. Remote Friendly：所有文件皆存放於 Notion，任何成員皆可隨時查閱。
6. Code Review Required：所有程式皆須透過 **Pull Request** 審核後才能合併。
7. Development：所有開發環境統一使用 Docker。
